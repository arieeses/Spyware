# 部署文档

架构:**一台中央主控** + **各面板服务器上的被控探针**。中央跑 Web 面板,探针上报日志/负载。

```
        中央主控(本项目, 1 台 VPS)
        ├─ Web 面板(登录/风控/下发)
        ├─ 连 v2board MySQL(只读, 拿用户画像+节点)
        └─ 收探针上报(日志+VPS负载)
              ▲ 一键安装
   [面板服务器1..8] 各跑一个被控探针 + Nginx spyware 日志
```

---

## 一、中央主控部署

一台 Linux VPS,Python 3.8+。**用 aaPanel 部署见 §一-B,裸机见 §一-A。**

### 一-0. 一键安装(最省事)

拉代码后一行命令装好(自动装依赖 + 建 systemd 服务 + 开机自启 + 启动):
```bash
git clone https://github.com/arieeses/Spyware.git spyware && cd spyware
sudo bash install.sh          # 可选自定义端口: PORT=9000 sudo bash install.sh
```
装完后按提示在 aaPanel/宝塔加个网站反代到 `127.0.0.1:8787` 并上 HTTPS(见 §一-B 第 3 步),打开 `/register` 注册管理员。
> 私有仓库 clone 需你的 GitHub 凭据(PAT / deploy key)。之后 §一-A/§一-B 的手动步骤可跳过。

### 一-B. aaPanel 部署(推荐给宝塔用户)

主控是普通 http.server 进程(非 WSGI),用 aaPanel 的**「Supervisor 进程守护」**跑,**「网站反向代理」**套 HTTPS。

**1) 放代码**
aaPanel 文件管理器上传解压, 或「终端」:
```bash
cd /www/wwwroot
git clone https://github.com/arieeses/Spyware.git spyware
cd spyware && pip3 install pymysql
```
`python3 --version` 需 ≥ 3.8(低了用 App Store「Python 项目管理器」装新版, 命令换成它的 python 绝对路径)。

**2) Supervisor 进程守护**(App Store 装「Supervisor 管理器」→ 添加守护进程)
| 项 | 值 |
|----|----|
| 名称 | spyware |
| 运行目录 | `/www/wwwroot/spyware` |
| 启动命令 | `python3 -m spyware.web --host 127.0.0.1 --port 8787` |
| 进程数量 | 1 |
| 运行用户 | root |

**3) 网站 + HTTPS + 反代**
- 「网站」→ 添加站点(绑域名, 纯静态, 不用建库/PHP)。
- 站点 → SSL → Let's Encrypt 申请, 开强制 HTTPS。
- 站点 → 反向代理 → 目标 URL `http://127.0.0.1:8787`, 发送域名 `$host`。
- **关键**: 在该站点「配置文件」的反代 location 里手动加一行(否则探针安装命令用 http):
  ```nginx
  proxy_set_header X-Forwarded-Proto $scheme;
  ```

**4)** 打开 `https://你的域名` → 注册管理员。

> 注意: 8787 绑 127.0.0.1 无需开防火墙; spyware.db 写在项目目录(root 跑最省事); v2board 同机用 127.0.0.1:3306 只读账号。

### 一-A. 裸机(systemd + Nginx)部署

### 1. 拉代码(私有仓库)
```bash
# 用 Personal Access Token 或 deploy key
git clone https://github.com/arieeses/Spyware.git /opt/spyware
cd /opt/spyware
pip3 install pymysql          # 唯一依赖(v2board 连接器用)
```

### 2. systemd 服务(绑 127.0.0.1)
```ini
# /etc/systemd/system/spyware.service
[Unit]
Description=Spyware central
After=network.target

[Service]
WorkingDirectory=/opt/spyware
ExecStart=/usr/bin/python3 -m spyware.web --host 127.0.0.1 --port 8787
Restart=always

[Install]
WantedBy=multi-user.target
```
```bash
systemctl daemon-reload && systemctl enable --now spyware
```

### 3. Nginx 反代 + HTTPS
面板自带登录但无 TLS,生产务必套 HTTPS 反代:
```nginx
server {
    listen 443 ssl;
    server_name spy.example.com;
    ssl_certificate     /etc/letsencrypt/live/spy.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/spy.example.com/privkey.pem;
    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;   # 探针据此用 https
    }
}
```
> `X-Forwarded-Proto` 必须传,否则探针一键安装命令会用 http。

### 4. 首次使用
浏览器打开 `https://spy.example.com` → 注册管理员账号。

## 二、接入 v2board 前端面板

每个 v2board 建一个**只读**数据库账号:
```sql
CREATE USER 'spy_ro'@'%' IDENTIFIED BY '强密码';
GRANT SELECT ON v2board.* TO 'spy_ro'@'%';
FLUSH PRIVILEGES;
```
面板「前端面板」→ 添加,填 host/端口/库名/只读账号/表前缀 → 点「同步」。
同步会自动读用户画像 + 节点表(协议、直连/中转、show、host)。

## 三、面板服务器接入(被控探针)

### 1. Nginx 订阅日志格式(每台面板机)
在订阅路由的 Nginx 配置加:
```nginx
log_format spyware '$time_iso8601|$http_x_forwarded_for|$remote_addr|'
                  '$status|$request_time|$http_user_agent|'
                  '$arg_token|$request_uri';

# CDN/反代场景必配, 还原真实 IP
set_real_ip_from <你的CDN网段>;
real_ip_header X-Forwarded-For;
real_ip_recursive on;

location /api/v1/client/subscribe {
    access_log /www/wwwlogs/spyware_sub.log spyware;
    proxy_pass http://v2board_backend;
}
```

### 2. 装探针(一键)
中央「日志接入」→ 添加「探针接入」→ 点「复制一键安装命令」→ 在面板服务器执行:
```bash
curl -fsSL "https://spy.example.com/agent/install.sh?token=xxx" | bash
# 日志路径非默认时: LOG=/你的路径 前置, 如
# curl -fsSL "..." | LOG=/www/wwwlogs/spyware_sub.log bash
```
探针装成 systemd 服务,自动增量读日志 + 采集 VPS 负载,按中央设的间隔上报。上报间隔在该源行的「自动」间隔里改,探针自动跟随。

## 四、下发(订阅网关,可选)

要做分级下发(高危/内鬼走隔离域名),订阅网关对每次拉取调:
```
GET https://spy.example.com/api/decision?token=<用户token>
→ {panel, tier, scope, domains:{协议:域名}}
```
按 `scope`(默认 relay=只改中转节点)重写订阅节点 host。详见面板「入口域名」页说明。

## 五、运维

- **备份**:定期备份 `/opt/spyware/spyware.db`(含配置/管理员/数据)。系统设置页可一键下载。
- **更新**:`git pull` 重启,或面板「系统设置 → 检查更新」一键升级(git 部署)。
- **安全**:面板只绑 127.0.0.1 + HTTPS 反代;`/api/ingest`、`/api/agent/*`、`/api/decision` 建议用防火墙限制来源(仅面板服务器/网关可达)。
- **迁移**:SQLite 单文件,拷 `spyware.db` 到新机,或设 `SPYWARE_DB=/路径` 环境变量指向它。
