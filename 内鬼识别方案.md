# 内鬼识别方案 · 总结与建议

> v2board 机场 · 识别「拉取订阅后攻击节点」的内鬼

---

## 1. 需求（已校正）

不是抓分销/共享,而是抓**合法订阅用户拉到节点清单后,对节点发起攻击/探测**(意图搞垮节点或让其被墙)。本质是**攻击归因**:已知某节点在某时被攻击 → 反查是哪个 token 干的。

## 2. 硬约束

**不能改动现有用户的订阅**(注水印/换节点会导致大量用户掉线)。→ 放弃"唯一化水印分发",改用**被动信号 + 事件求交**归因。

## 3. 内鬼画像（四信号先验 + 流量背离）

正常用户与内鬼的判别,靠**组合**而非单条:

| 维度 | 正常用户 | 内鬼 |
|------|---------|------|
| **ASN**（最强） | 住宅 / 移动运营商 | **机房 / IDC / VPS** |
| **UA** | 已知客户端(clash/v2rayN/Shadowrocket/sing-box…) | 工具类(curl/wget/python-requests/Go-http-client)/空/伪造 |
| **拉取频率** | 零散(启动+定时更新) | 机器般规整 / 异常高频 |
| **流量背离**（抗伪装） | 有拉取 **且** 有真实流量 | **持续拉取 却 流量≈0**(只收集节点、不使用) |
| **拉取后IP静默**（高分辨率） | 拉取IP 随即出现在节点连接中 | 拉取IP 拉完就"不通"/**从不连节点**(grab-and-go) |
| **TLS/HTTP指纹**（抗伪装·最强） | 真客户端固定 JA3/JA4 | 工具指纹(curl/Go/python),或 **UA与指纹矛盾**=伪造 |
| **注册→拉取轨迹**（新号判别） | 注册→拉取→该IP随即连节点 | 注册→立即拉取→该IP不通/从不连节点=侦察号 |

**判据铁律**:任何单条信号都不定罪。锁定嫌疑靠组合:
`机房ASN + 工具UA + 规整高频 + 拉取活跃却零流量背离`。

**流量背离要点**:抓的是「**拉取活跃 × 流量为零 的背离**」,不是"零流量"——否则误伤休眠/囤号(他们不怎么拉)。新号不用账龄一刀切放过,改用**注册→拉取→使用轨迹**(见下条);用"背离度"扛住"故意跑点流量"的伪装。数据来自 v2board `v2_stat_user`,零采集成本。

**注册→拉取轨迹要点**(修正"账龄宽限期"):新号不是天然无辜。`注册→拉取→该IP随即连节点=正常`;`注册→立即拉取→该IP不通/从不连节点=侦察号`(注册就为拿节点清单)。串起 `v2_user.created_at × 拉取时间 × 节点侧该IP有无连接` 三份数据判轨迹,比账龄宽限准得多,对新号尤其有效。

**拉取后IP静默要点**(流量背离的 IP 级+时间级高分辨率版):对每次拉取 `(token, IP_pull, t_pull)`,看 IP_pull 是否在其后窗口出现在**节点连接记录**里。指标:①拉取后静默时长(始终不连=完全静默);②完全静默占比。高占比+持续 → 嫌疑。比账号级零流量更难规避——它暴露的是"采集行为本身"。
- **前提**:必须有**节点侧按 IP 连接记录**(Xray/sing-box access log 或 v2board 在线 IP),否则判断不了"拉完用没用"。
- **误判规避**:NAT/多出口 → 匹配放宽到 **/24 或 ASN**;客户端定时更新订阅但没上网属正常 → 看**持续性**,别用单次短静默定罪;与流量背离**同向**时置信度乘积上升。
- **时间段**:统计"只拉不连"集中的时段,可暴露自动化采集窗口。

## 4. UA 白名单放行 · 铁律

放行是**降噪工具,不是安全边界**。UA 可任意伪造,必须 gated by ASN:

| UA | ASN | 结论 |
|----|-----|------|
| 白名单客户端 | 住宅/移动 | ✅ 真放行 |
| 白名单客户端 | **机房** | 🔴 **视为伪造,最高优先级可疑** |
| 非白名单/工具/空 | 任意 | ⚠️ 进嫌疑池 |

实现:白名单用**前缀+版本通配正则**、**default-deny**、未匹配 UA 样本落库人工审查。只在**分析层降噪**,慎在 Nginx 接入层硬拦(误伤+UA易伪造)。

## 4.2 增强识别率的补充信号（按价值排序）

识别率提升来自两个方向:**①抗伪装的指纹类信号**(让伪装失效→提召回) + **②跨账号/维度关联**(单点变集群)。

1. **TLS/HTTP 指纹(JA3/JA4)—— 最强,废掉 UA 伪装**
   UA 可改,但 TLS 握手指纹由客户端 TLS 库决定、改不了。真 Clash/Shadowrocket/sing-box 各有固定 JA3/JA4;curl/python/Go 完全不同。
   - **杀招·矛盾检测**:UA 声称 Shadowrocket(iOS) 但 TLS 指纹是 Go/Linux → 近乎铁证的伪造。抓住所有"伪造 UA 骗过白名单"的内鬼。
   - 落地:Nginx/反代层记录 JA3/JA4(nginx-ssl-ja3 或前置可出指纹的代理),与 UA 做一致性校验。

2. **HTTP 请求头 + 行为完整性**
   - 请求头指纹:真客户端 `Accept/Accept-Encoding/Accept-Language` 及 **header 顺序**固定;脚本是极简/默认头。
   - 行为完整性:真客户端除 `/subscribe` 还打其他 API;**纯采集者只薅订阅端点、别的不碰**。

3. **跨 token 关联(一个内鬼 = 多马甲)—— 提召回**
   用同一拉取 IP / 同一 TLS 指纹 / 同一设备特征把多个 token 串起来。抓到一个 → 顺藤摸出所有马甲整簇端掉;"多账号共享同一异常指纹"本身即强嫌疑。

4. **注册/账号维度(源头筛)**
   临时邮箱域名、注册 IP 是机房 ASN、免费/试用/最低档套餐、注册后立即只拉不用、批量注册/同优惠码/同支付源多号。侦察号注册时就露馅。

5. **使用形态:扫描式短连 vs 正常长连(需节点侧数据)**
   正常用户长连少数节点持续上网;侦察者**对每个节点各连一下就断**(探活),"遍历所有节点、每个只碰一次" = 采集行为。即使跑了点流量伪装也能区分。

6. **无监督聚类/离群检测(系统性提召回)**
   积累特征向量后加一层 Isolation Forest 类离群检测,捞出"明显偏离正常群体、但没命中任何现成规则"的账号复核——专抓规则漏掉的新型内鬼。

7. **威胁情报沉淀 + 跨机场共享**
   确认内鬼的指纹(JA3/IP/ASN/邮箱域名)沉淀为黑名单,新号命中即高危;多机场纳管时**黑名单跨机场共享**——A 机场被抓,B 机场直接现形。

> 优先级:先上 **①TLS/HTTP指纹**(最大单点增益,让 UA 信号从可绕过变难伪造),再 **④注册维度**(零成本源头拦截) + **③跨token关联**(提召回);⑥⑦为后期系统性增强。

## 4.5 排除层 · 评分前先摘掉「确定正常」

评分和归因之前先跑一个**排除层**,降误判 + 降计算量。两类:

1. **自有 IP 白名单**(减少 ASN 误判的关键):你自己的基础设施会从机房 IP 拉订阅——subconverter/订阅转换、监控/健康检查/测速、节点服务器、v2board 面板、备份任务——它们恰好三信号(机房ASN+工具UA+高频)全中却是正常的。命中自有 IP → 排除。
   - **铁律**:只白名单**你自己的具体 IP / CIDR**,**绝不白名单整个 ASN**(否则同机房攻击者免检)。
   - **维护**:从**节点清单自动同步**,加/删节点白名单跟着更新。
   - **nuance**:建议对自有 IP 的拉取**单独记账**而非完全丢弃,防节点被入侵后拉取被隐藏。
2. **客户端 UA 白名单 + 住宅/移动 ASN** → 确定正常,降噪(见 §4)。

## 5. 归因核心 · 攻击事件时间窗求交

不改订阅也能收敛到人的关键方法:

1. 记录每次订阅拉取:`token、时间、来源IP/ASN/UA、拉到的节点集`。
2. 每次节点攻击(时间 T,目标节点 X),取嫌疑集
   `S_i = { 攻击前 Δ 时间内拉过含 X 订阅的 token }`。
3. **内鬼必然出现在每次攻击的 S_i 中**,无辜用户随机进出。
4. 多次事件**求交** `S_1 ∩ S_2 ∩ … ∩ S_n` → 嫌疑集指数收缩,几次事件从上千人收敛到个位数。
5. 先用**流量背离**把嫌疑池预筛成"沉默拉取者",再求交 → 又快又准。

优点:归因基于"谁具备被攻击节点的可见性",不依赖攻击源 IP → 内鬼换 IP/挂代理都无效;且**只读日志,不碰订阅**。

## 6. 可选增强（不破坏任何用户）

不能"改"节点,但"**新增**"节点是加法、不掉线:定期**新增/轮换节点**,攻击者要打新节点就必须重新拉订阅 → 每个新节点的可见性集极小。极致做法:**增量金丝雀节点**(只放进部分用户订阅的新节点),被攻击即锁定这部分人——不违反约束,威力接近水印法。

## 7. 数据源（全部只读）

- **Nginx 订阅路由 access log**:token、时间、IP、UA(四信号来源)
- **v2board `v2_stat_user`**:每用户流量(流量背离)
- **节点侧攻击事件**:被攻击节点、时间、攻击特征(先能"检测到被攻击")
- **ASN 分类库**(住宅/机房)+ IP→ASN 映射（准确性地基）

## 8. 建议（落地优先级）

1. **先建两样地基**:①ASN 分类库;②订阅拉取记录(token+时间+ASN+UA+节点集)。没有这两样,一切无从谈起。
2. **先做"流量背离"先验**:成本最低(数据现成)、抗伪装最强,直接产出"沉默拉取者"高嫌疑名单。
3. **叠加 ASN+UA+频率组合先验**,给嫌疑排序;UA 白名单只做降噪。
4. **上归因引擎**(攻击事件时间窗求交),用先验预筛后求交,跨事件累积置信。
5. **补增量金丝雀节点**(可选),把归因从"高置信"推向"铁证"。

**诚实边界**:纯被动+不改订阅,**单次攻击做不到铁证**;准确度来自**多事件累积**。系统要设计成"持续观测、跨事件累积置信",而非幻想一次抓准。处置前保留**证据链**(嫌疑集+异常指纹+事件记录),先隔离观察、再定性。
